# W-15 — Defamation exposure on /verify

| Field | Value |
|---|---|
| ID | W-15 |
| Severity | **High** |
| Status | **fix proposed** |
| Authoritative ref (in TRUTH or repo) | TRUTH.md Section G |
| Source(s) in original docs | SRD §27.9, SRD §28.3 |

## Problem

Even publishing a *commitment hash* tied to an IPFS-pinned dossier risks defamation if the hash + side-channel reveals an entity name. SRD §28.3 admits 'we do not publish allegations against named individuals' but the architecture allows /verify to reveal more than intended.

## Fix

**Counsel review required** (counsel pre-retained per EXEC §16). Minimum viable answer: /verify publishes only the audit-chain root + escalation-event metadata (date, council vote tally, dossier number). Entity names appear ONLY in the dossier PDF, which is delivered to CONAC and never auto-pinned to public IPFS without explicit council 4-of-5 vote (higher bar than 3-of-5 for escalation).
