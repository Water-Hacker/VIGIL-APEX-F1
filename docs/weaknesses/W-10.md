# W-10 ‚Äî WebAuthn ‚Üí secp256k1 path is fragile

| Field                                | Value                                                                |
| ------------------------------------ | -------------------------------------------------------------------- |
| ID                                   | W-10                                                                 |
| Severity                             | **Medium**                                                           |
| Status                               | **fix shipped (cryptographic core); desktop wrapper deferred M3-M4** |
| Authoritative ref (in TRUTH or repo) | TRUTH.md Section E                                                   |
| Source(s) in original docs           | SRD ¬ß17.8.3                                                          |

## Problem

SRD ¬ß17.8.3 attempts COSE-alg-`-47` (secp256k1) translation with 'trial recovery' of the v byte. Most browsers do not surface secp256k1 in WebAuthn (Chrome supports it on YubiKey 5+ since firmware 5.4, Firefox/Safari support is variable).

## Fix

**Default path: native desktop helper** (Tauri or Electron-with-strict-CSP) that uses `libykcs11` directly to drive the YubiKey for council vote signing. Reserve WebAuthn for accessibility-edge cases. Council members install the helper at the ¬ß13 enrolment ceremony; the architect publishes a reproducible build with EV signature.

WebAuthn fallback remains documented for rare browser-only scenarios but is not the primary path.

## Status ‚Äî 2026-05-17

The cryptographic core of the native path is shipped:

- `tools/vigil-council-signer/rust-helper/` ‚Äî Rust binary that drives
  the YubiKey PIV P-256 key via PKCS#11 (`cryptoki` crate),
  decodes DER signatures, low-S normalises, and self-verifies the
  signature against the token's own public key before returning.
  Pure helpers (DER decode, normalise, point parse, verify) are
  proptest-fuzzed.
- `tools/vigil-council-signer/main.py` ‚Äî per-user NDJSON-over-Unix-
  socket service that wraps the Rust helper. Mirrors the polygon-
  signer wire-protocol shape so an SRE only learns one schema.
- `infra/systemd/vigil-council-signer.user.service` ‚Äî systemd-user
  unit (NOT --system; the YubiKey lives on the member's workstation,
  not the central host).
- `packages/security/src/council-signer.ts` ‚Äî Node-side bridge
  (`isCouncilSignerAvailable`, `getCouncilPubkey`,
  `signCouncilChallenge`). The dashboard's council-vote flow probes
  the socket; native path used when present, WebAuthn fallback
  otherwise. 17 vitest cases pin every documented branch.

Deferred to M3-M4 (per W-10 spec): Tauri / Electron desktop wrapper
embedding this service, with an EV-signed reproducible build the
architect publishes for every release. The cryptographic boundary is
already done; the wrapper is a UX shell around it.
