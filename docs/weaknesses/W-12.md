# W-12 — Vault Shamir storage on YubiKey challenge-response slot mis-specified

| Field | Value |
|---|---|
| ID | W-12 |
| Severity | **High** |
| Status | **fix proposed** |
| Authoritative ref (in TRUTH or repo) | TRUTH.md Section E |
| Source(s) in original docs | HSK §5.10 |

## Problem

HSK §5.10 stores Vault Shamir share via challenge-response (HMAC-SHA1). HMAC is a MAC, not an encryption primitive — you cannot retrieve the share from the YubiKey, only verify a challenge against it. This is a security-correctness bug.

## Fix

**Switch to age-plugin-yubikey**: each share is encrypted to the holder's YubiKey PIV slot 9d (Card Authentication) using EC-DH key agreement (`age` + `age-plugin-yubikey`). Decryption requires the YubiKey + PIN; the encrypted blob lives on disk or in the safe envelope.

Paper backups of the share remain unchanged (alternative recovery path).

`age` and `age-plugin-yubikey` are committed to the dev tooling in §4.4 of the assimilation plan.
