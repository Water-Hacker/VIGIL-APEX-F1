# W-09 — Tip portal is Tor-friendly not Tor-native

| Field | Value |
|---|---|
| ID | W-09 |
| Severity | **High** |
| Status | **fix proposed** |
| Authoritative ref (in TRUTH or repo) | TRUTH.md Section F |
| Source(s) in original docs | SRD §28, SRD §28.4 |

## Problem

SRD §28.4 gives client-side encryption + IP-log purging for the /tip portal but does NOT include a `.onion` v3 hidden service. Cameroonian whistleblowers under threat need a hidden service that does not require a Tor exit node.

## Fix

**Stand up a `.onion` v3 hidden service** from M3 day-1 (Phase 3 delivery plane). Same Next.js app, served by a separate Tor onion-balanced front-end on N02. Documented on the public /tip page with a copy-pasteable `.onion` address. In-country submitters reach via Tor + obfs4 bridge to defeat ISP fingerprinting (per W-13 layered egress policy).
