# W-01 — Host OS conflict — Ubuntu+LUKS2 vs Windows+BitLocker

| Field | Value |
|---|---|
| ID | W-01 |
| Severity | **High** |
| Status | **fix committed** |
| Authoritative ref (in TRUTH or repo) | TRUTH.md Section B |
| Source(s) in original docs | SRD §02.3, HSK §4.1 |

## Problem

SRD §02.3 specifies Ubuntu Server 24.04 LTS + LUKS2 + Btrfs subvolumes on the MSI Titan. HSK §4.1 specifies Windows + BitLocker + NTFS on the same machine. Both cannot be true.

## Fix

**Pick Ubuntu 24.04 LTS + LUKS2 + clevis-YubiKey + Tang on Synology**. Rationale: matches the Docker/systemd architecture in SRD §02-§04; avoids the BitLocker-via-PIV smartcard fragility documented in HSK §4.1; preserves the host-as-N01 model in SRD §02.

HSK §4.1 will be rewritten in `docs/source/HSK-v1.md` from BitLocker semantics to LUKS2-clevis-YubiKey semantics. The mechanism is essentially identical (YubiKey-PIV-protected pre-boot disk unlock); only the underlying tooling differs.
