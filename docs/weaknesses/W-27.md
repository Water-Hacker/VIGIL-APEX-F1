# W-27 — Decision log not CI-enforced

| Field | Value |
|---|---|
| ID | W-27 |
| Severity | **Low** |
| Status | **fix proposed** |
| Authoritative ref (in TRUTH or repo) | OPERATIONS.md §7 |
| Source(s) in original docs | EXEC §37 |

## Problem

EXEC §37 specifies `/docs/decisions/log.md` but no enforcement that all FINAL decisions get audit_event entries at Phase 1.

## Fix

**`scripts/check-decisions.ts` lint** runs on every push to `main` and fails the build if:
- Any decision marked `Status: FINAL` is dated within the last 7 days but lacks a corresponding `audit_event_id` (Phase 1+ only).
- Any decision-log entry references a phase or section that does not exist.

Lint shipped in Phase 0 scaffold.
