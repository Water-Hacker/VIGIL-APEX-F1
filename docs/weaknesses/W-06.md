## Problem

PPTX promises '10 federated regions, ZK-Proofs, decentralised, no single off-switch'. SRD/MVP delivers 1 host + 1 VPS + 2 NAS hybrid. There is no documented Phase-2 → Phase-N path that takes the system from 1 region to 10. This is the **biggest documentation problem in the pack**.

## Fix

**Add explicit `ROADMAP.md`** with phase semantics: Phase 1 = MVP pilot, Phase 2 = production hardening + MOU-gated escalation, Phase 3 = regional federation, Phase 4 = full 10-region with ZK-proofs. Mark every PPTX claim that exceeds Phase 1 with a phase tag.
